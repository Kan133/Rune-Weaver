# Server Entry 与项目结构

## 1. 用途

解决 Rune Weaver 生成的服务端代码如何正确接入 Dota2 项目的问题。明确项目结构和入口点。

## 2. 上游来源

- `references/dota2/docs/moddota_scripting_typescript.md`
- 章节: "Typescript Introduction"
- 关键页面: https://moddota.com/scripting/Typescript/typescript-introduction

## 3. 核心要点

### 3.1 x-template 项目结构

```
game/scripts/src/
├── modules/
│   └── index.ts           # 宿主模块入口 (inject_once 目标)
├── addon_game_mode.ts     # 游戏模式入口 (宿主原有)
└── rune_weaver/           # RW 命名空间 ✅
    ├── index.ts           # RW 总入口 (create)
    └── generated/
        ├── server/
        │   ├── index.ts   # Server 索引 (refresh)
        │   └── {feature}.ts  # feature 文件
        └── shared/
            ├── index.ts   # Shared 索引 (refresh)
            └── {feature}.ts  # feature 文件
```

### 3.2 关键入口文件

**宿主模块入口** (`modules/index.ts`):
```typescript
// 宿主原有代码...
import { activateRuneWeaverModules } from "../rune_weaver";

export function ActivateModules() {
  // 宿主原有初始化...
  
  // RW 桥接点
  activateRuneWeaverModules();
}
```

**RW 总入口** (`rune_weaver/index.ts`):
```typescript
// Generated by Rune Weaver
import { activateRwGeneratedServer } from "./generated/server";

export function activateRuneWeaverModules(): void {
  activateRwGeneratedServer();
}
```

**Server 索引** (`rune_weaver/generated/server/index.ts`):
```typescript
// Generated by Rune Weaver - Refresh on demand
import { registerRwDashQ } from "./rw_dash_q";
import { registerRwTalentDraw } from "./rw_talent_draw";

export function activateRwGeneratedServer(): void {
  registerRwDashQ();
  registerRwTalentDraw();
}
```

### 3.3 TypeScript 编译流程

```
src/vscripts/abilities/{feature}.ts
    ↓ (tstl 编译)
game/scripts/vscripts/abilities/{feature}.lua
```

**Rune Weaver 职责**:
- 生成 `.ts` 文件到 `src/vscripts/` (或等效目录)
- 不直接操作 `.lua` 文件
- 依赖宿主构建流程完成编译

### 3.4 文件生成位置

| 抽象目标 | 生成路径 | 编译后路径 |
|----------|----------|------------|
| server | `src/rune_weaver/generated/server/{id}.ts` | `game/scripts/vscripts/rune_weaver/generated/server/{id}.lua` |
| shared | `src/rune_weaver/generated/shared/{id}.ts` | `game/scripts/vscripts/rune_weaver/generated/shared/{id}.lua` |

注意：具体路径可能因宿主模板而异，x-template 使用 `game/scripts/src` 作为 TypeScript 源码目录。

## 4. 对 Rune Weaver 的直接价值

| 场景 | 价值 |
|------|------|
| Host Mapping | 明确服务端文件落点 |
| Bridge Planning | 明确 `modules/index.ts` 是 inject_once 目标 |
| Write Executor | 知道生成文件应该放在哪里 |
| Host Validation | 检查命名空间目录是否存在 |

## 5. 当前最相关的 Pattern / Module

- `effect.dash` → 生成到 `generated/server/`
- `effect.modifier_applier` → 生成到 `generated/server/`
- `rule.selection_flow` → 生成到 `generated/server/`
- `input.key_binding` → 生成到 `generated/server/`

## 6. 后续注意事项

- 始终通过 `activateRuneWeaverModules()` 桥接，不要直接修改 `addon_game_mode.ts`
- 生成的文件需要符合 tstl (TypeScript-to-Lua) 语法
- 避免使用 TypeScript 特有语法（如枚举在 tstl 中有特殊处理）
- Shared 目录用于 Server 和 UI 共享的类型定义
