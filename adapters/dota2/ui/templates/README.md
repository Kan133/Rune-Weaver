# Dota2 UI Templates

This directory contains reference templates for Dota2 Panorama UI surfaces.

Current status:

- Templates document the intended component and style shape.
- The active generator may still emit TSX/LESS through dedicated generator modules instead of rendering these templates directly.
- Generator output must still satisfy [UI-SAFER-PROFILE.md](../../../../docs/UI-SAFER-PROFILE.md).

## Template Responsibilities

Templates may define:

- component structure,
- stable CSS class names,
- pattern parameter placeholders,
- `UIDesignSpec` token placeholders,
- safe fallback slots.

Templates must not define:

- business rule logic,
- server-side effect application,
- key binding registration,
- host bridge mutation,
- arbitrary custom UI generated by an LLM.

## LESS Handling

For x-template Panorama targets:

- Do not import generated LESS from generated TSX.
- Add generated LESS imports to `content/panorama/src/hud/styles.less`.
- Style import insertion must be idempotent.

## Selection Modal Safety Contract

`selection-modal` templates and generator output must support:

- `payloadShape`,
- `minDisplayCount`,
- `placeholderConfig`,
- disabled placeholder cards,
- stable card dimensions,
- defensive payload normalization,
- cleaned-up event subscriptions,
- confirm disabled until a selectable item is selected.

## Template Variable Rules

Every template variable must be traceable to one of:

- pattern parameters,
- `UIDesignSpec`,
- controlled generator defaults,
- host adapter constants.

Do not add free-form template variables that require an LLM to invent runtime behavior.
