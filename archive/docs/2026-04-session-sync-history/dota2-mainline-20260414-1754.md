   - ready to apply

   - blocked by policy
2. Add a clearer one-click continuation rail after successful apply:
   - repair + build

# Current blockers / risks

- The lifecycle contracts are now in place, but the Workbench panel still needs copy/state polish before it feels fully product-grade.
- Pre-write readiness in Workbench is currently surfaced through the structured bridge review payload, not a dedicated standalone CLI command/result shape yet.
- The current apply path still patches repo-side generator anchor files, which is the intended scope. Other sessions should not describe this as arbitrary host/runtime patching.
- There is still a product gap between “approval/apply exists” and “the approval unit is unmistakably understandable to a novice user.”
